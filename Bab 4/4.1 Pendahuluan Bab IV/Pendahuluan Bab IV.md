# Pendahuluan Bab IV

Bab ini menyajikan **hasil penelitian** setelah prosedur pada Bab III dilaksanakan. Pada penelitian pengembangan perangkat lunak dengan metode Scrum, hasil disusun mengikuti **urutan artefak dan alur iterasi**: mulai dari capaian *Sprint Backlog*, pemantauan harian *Daily Scrum*, pengujian *increment*, tinjauan *Sprint Review*, refleksi *Sprint Retrospective*, hingga kelayakan *Shippable Product*, dan ditutup dengan **hasil evaluasi kebergunaan** menggunakan *System Usability Scale*.

Susunan subbab pada Bab IV diselaraskan dengan struktur folder pada direktori `Bab 4`, yaitu:

| Subbab | Isi ringkas |
| --- | --- |
| 4.1 | Pendahuluan Bab IV |
| 4.2 | Hasil *Sprint Backlog* |
| 4.3 | Hasil *Daily Scrum* |
| 4.4 | Hasil *Increment Development & Testing* |
| 4.5 | Hasil *Sprint Review* |
| 4.6 | Hasil *Sprint Retrospective* |
| 4.7 | Hasil *Shippable Product* |
| 4.8 | Hasil evaluasi penelitian dan lampiran kuesioner SUS |

Pembahasan interpretatif terhadap temuan, keterbatasan, dan implikasi dapat Anda letakkan pada subbab tersendiri di akhir Bab IV atau pada Bab V sesuai pedoman program studi. Dokumen pada setiap folder dapat disalin ke naskah utama Word dengan penomoran subbab yang disesuaikan.
